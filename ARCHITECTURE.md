## Слои системы

| Слой | Ответственность |
|---|---|
| **API** | Единый контракт: проекты, анализы, кампании, память бренда, прогноз, демо |
| **Сервисы** | Сбор цифрового следа, память бренда, агентный цикл, бенчмарки, атрибуция |
| **LLM-шлюз** | Унифицированный OpenAI-совместимый интерфейс → российский стек (смена провайдера параметром) |
| **Хранилище** | Реляционная БД + векторный слой для семантического поиска по памяти бренда |
| **Фронтенд** | Веб-интерфейс: память бренда, цифровой след, бенчмарки, режимы, демо, атрибуция |

## Ключевые архитектурные решения

- **Память бренда как долгоживущий актив.** Не stateless-генератор, а накапливаемая
  база знаний на каждый бизнес — с временной осью. Главный источник ценности.
- **Сходимость вместо параллельных промптов.** Агентный цикл повторяется до достижения
  целевой экономики (прогноз CPL), а не выдаёт разовый результат. Это защищаемая часть.
- **Импортонезависимость.** Продакшн-инференс изолирован за LLM-шлюзом; продукт не
  привязан к зарубежным моделям.
- **Два режима работы:** стратегический (месячный план, несколько гипотез с прогнозом)
  и тактический (разовая задача под канал за минуты).

## Инженерные принципы

- Обратимые изменения малыми срезами, каждый — с проверкой работоспособности.
- Честный статус: в документации отражается, что реально работает, а что — план.
- Никаких обещаний в демо того, чего нет в коде.
