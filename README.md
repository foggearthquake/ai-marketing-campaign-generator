## Что это

**ИИ-Кампания** — не «ещё один генератор текстов». Это **AI-маркетинговый директор**:
он изучает бизнес, рынок, конкурентов и город, ведёт живую память о бренде и каждую
неделю говорит владельцу, что делать с рекламным бюджетом, чтобы он окупился —
с прогнозом стоимости лида (CPL) **до** запуска кампании.

> *«У большого бизнеса есть маркетинговый отдел. У малого — теперь есть мы.»*

**Почему «директор», а не «генератор»:** генератор обесценивается до ChatGPT.
Позиция маркетинг-директора в малом бизнесе пустует (штатник стоит ~150 000 ₽/мес,
фаундер не потянет). Продукт занимает эту позицию за 990–9 990 ₽/мес.

**Статус:** выигран грант Фонда содействия инновациям «Студенческий стартап»
(1 000 000 ₽). Ядро MVP собрано и проверено сквозным прогоном (см. ниже).

## Архитектура — 4 модуля

| Модуль | Назначение | Суть |
|---|---|---|
| **01 · Цифровой след** | Сбор контекста о бизнесе | Расширяет вход с одного URL до множества источников (сайт, документы, конкуренты, отзывы, спрос, события города) за минуты параллельной обработки |
| **02 · Память бренда** *(ядро)* | Живая база знаний компании | Своя БД на бизнес: продукт, портреты ЦА, голос клиента, карта конкурентов, история кампаний с метриками, гипотезы, сезонность. Диалог по своим данным (гибрид direct/RAG) |
| **03 · Агентный цикл** *(инновация)* | Генерация со сходимостью | Не «5 параллельных промптов», а цикл Стратег → Исследователь → Креативщик → Критик с реальным tool use и **сходимостью по CPL** |
| **04 · Прогноз CPL** | Экономика до запуска | Прогноз CPL/CAC/ROI/ROAS диапазоном + светофор риска 🟢/🟡/🔴. Клиенты пополняют базу бенчмарков → сетевой эффект |

Подробная схема потоков данных — в [`ARCHITECTURE.md`](ARCHITECTURE.md).

## Технологический принцип — импортонезависимость с дня 1

- Продакшн-инференс — на **российском стеке** (YandexGPT, GigaChat) через
  унифицированный OpenAI-совместимый шлюз: смена провайдера — один параметр.
- **Никакой зашитой зависимости** от OpenAI / Anthropic / Google в рантайме продукта.
- Backend: FastAPI / Python. Frontend: Next.js / React. Хранилище: PostgreSQL + pgvector.

Это не просто техрешение — это **позиционный ров** (важно для грантовой и корпоративной
среды РФ).

## Инженерный процесс — из чего складывалась работа

Проект не писался «с нуля начисто». Это была дисциплинированная **пересборка**:

1. **Честный аудит** существующего кода против продуктового видения — что реально
   работает, что заглушка, что сломано (зафиксировано документально).
2. **Архитектурная хирургия** малыми обратимыми срезами: вырезан тяжёлый SaaS-каркас
   (мультитенант, биллинг, очереди, auth), который был построен «на 0 клиентов» —
   каждый срез с проверкой, что приложение остаётся рабочим.
3. **Достройка 4 модулей** поэтапно, от ядра (Память бренда) к периферии.
4. **Инженерная гигиена:** миграции БД, типизация, линтеры, pre-commit, CI,
   метрики, структурные логи — то, чего в исходной версии не было.

Дисциплина: только необходимые изменения, обратимость через архив, статическая
проверка импортов, и — никаких обещаний в демо того, чего нет в коде.

## Доказательство работоспособности

Ядро MVP проверено **сквозным прогоном** `URL бизнеса → 3 кампании с прогнозом CPL`:

```
URL → Цифровой след → Память бренда → Агентный цикл (tool use, сходимость по CPL)
    → 3 гипотезы кампаний: angle + объявления + email-цепочка + структура лендинга
    + прогноз CPL + светофор риска
```

- Автотесты ядра: **89 passed** ✅
- Приложение поднимается, сквозной демо-сценарий отрабатывает и возвращает
  готовые кампании на русском с оценкой экономики.

> Скриншоты / запись демо — по запросу. Рабочий код, реализующий этот прогон,
> в публичный доступ намеренно не выкладывается.
